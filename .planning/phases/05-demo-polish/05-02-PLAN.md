# Plan 05-02: Seed KB Script — Pre-populate Patterns at Multiple Confidence Tiers

## Goal

Create `scripts/seed_kb.py` — a standalone Python script that pre-populates the AKC knowledge base with 30 synthetic patterns across Gold (5), Production (10), and Experimental (15) tiers. The script writes directly to `patterns.jsonl` using the same `Pattern` model from Phase 1 (`akc.patterns.models`) for validation, ensuring seeded data is structurally identical to live-learned patterns.

## threat_model

**ASVS L1 Threat Assessment**

| Threat | Vector | Mitigation |
|--------|--------|------------|
| Path traversal via `--kb-dir` | Attacker passes `../../etc` | Script uses `Path(kb_dir).resolve()` — absolute path used in all operations. No shell expansion. Script is a local admin tool, not an HTTP endpoint. |
| Overwrite attack — existing data destroyed | Script run without `--overwrite` flag | `FileExistsError` raised if `patterns.jsonl` exists and `--overwrite` not set. Clear error message with remediation. |
| Import of malicious module | Fake `akc.patterns.models` on PYTHONPATH | Script runs in same venv as service; no dynamic import. User must have write access to venv to replace modules. |
| Synthetic data treated as real | Seeded patterns confused with live-learned patterns | Seeded patterns have same structure as live patterns (no synthetic flag). This is intentional — demo requires credible patterns. Documented in README. |
| Permissions error on volume mount | `/app/data/kb` not writable | Script catches `PermissionError` and prints clear message: `"Cannot write to {kb_dir} — ensure directory is writable"`. Exits 1. |

**Security controls enforced:**
- `Path(kb_dir).resolve()` — absolute path, no traversal
- `FileExistsError` guard unless `--overwrite` explicit
- No subprocess calls, no shell expansion
- All writes use `open(..., "a", encoding="utf-8")` — append-only consistent with store.py

## Context

**From research:** The seed script uses `Pattern` model from `akc.patterns.models` for pydantic validation (catches missing fields), then writes `pattern.model_dump_json() + "\n"` directly to `patterns.jsonl`. It does NOT use `JsonlStore.update_pattern` (which is for existing patterns) — instead it appends directly, consistent with Phase 1's `_write_patterns_atomic_sync` approach.

**Key Phase 1 contracts:**
- `Pattern` model fields: `id`, `context`, `what_worked`, `what_failed`, `tags`, `confidence`, `tier`, `consecutive_failures`, `times_applied`, `last_updated`
- `Pattern.id` defaults to `str(uuid.uuid4())` — script must NOT set id manually; let model generate it
- `tags` normalized to lowercase via `@field_validator` on `Pattern`
- `tier` is a `Tier` enum with values: `"gold"`, `"production"`, `"experimental"`, `"demoted"`
- Write to `{kb_dir}/patterns.jsonl` (same path as `JsonlStore`)

**INIT_CONFIDENCE from Phase 1:** 0.67. Seeded Experimental patterns use confidence range 0.50–0.70; seeded Gold/Production are above that range.

    **`seed_kb(kb_dir: str, patterns: list[dict], overwrite: bool) -> None`:**
    ```python
    def seed_kb(kb_dir: str, patterns: list[dict], overwrite: bool) -> None:
        path = Path(kb_dir).resolve()
        path.mkdir(parents=True, exist_ok=True)
        patterns_file = path / "patterns.jsonl"
        if patterns_file.exists() and not overwrite:
            raise FileExistsError(
                f"patterns.jsonl already exists at {patterns_file}. Use --overwrite to replace."
            )
        if overwrite and patterns_file.exists():
            patterns_file.unlink()
        try:
            with open(str(patterns_file), "a", encoding="utf-8") as f:
                for i, pattern_dict in enumerate(patterns, 1):
                    # Validate via Pattern model (catches missing fields)
                    from akc.patterns.models import Pattern
                    pattern = Pattern(**pattern_dict)
                    f.write(pattern.model_dump_json() + "\n")
                    if i % 10 == 0 or i == len(patterns):
                        print(f"  [{i:3d}/{len(patterns):3d}] patterns written")
        except PermissionError:
            raise PermissionError(f"Cannot write to {patterns_file} — ensure directory is writable")
    ```
