# Plan 05-01: SKILL.md — Claude Code Skill for Recall → Task → Remember Loop

## Goal

Create `.claude/skills/akc-recall-task-remember/SKILL.md` — a Claude Code skill that automates the full recall → task → remember loop. When invoked as `/akc-recall-task-remember`, the skill calls `POST /recall`, synthesizes a task solution from returned patterns, then calls `POST /remember` with the outcome and reports confidence changes via `GET /stats`.

## threat_model

**ASVS L1 Threat Assessment**

| Threat | Vector | Mitigation |
|--------|--------|------------|
| SSRF via `--endpoint` arg | Attacker passes internal endpoint URL | Skill is a local Claude Code tool — executes in user's environment; no server-side trust boundary. Document that `--endpoint` must be `localhost` or trusted host. |
| Command injection via `--task` | Shell-injected payload in task string | Python implementation uses `requests.post(url, json=payload)` — task string goes in JSON body, never shell-expanded. No `subprocess` or `os.system` calls. |
| Sensitive data in stdout | Patterns may contain internal system context | Skill prints to stdout only in terminal session. No logging to persistent files; no HTTP callbacks. |
| Credential exposure | `LLM_API_KEY` in env var | Skill reads `--endpoint` arg only; never reads env vars directly. AKC service holds credentials server-side. |
| Denial-of-service via large `--top-k` | `top_k=1000` floods recall response | Default `top_k=5`; requests library timeout `5.0s` on all calls. If /recall returns large payload, skill still processes in-memory. |

**Security controls enforced:**
- All HTTP calls use `timeout=5.0` (requests library)
- Task string goes into JSON body — never shell-expanded
- No file writes; no persistent secrets
- `raise_for_status()` on all responses — explicit error on 4xx/5xx

## Context

**From research:** SKILL.md is a Claude Code skill file registered at `.claude/skills/akc-recall-task-remember/SKILL.md`. The skill orchestrates three sequential HTTP calls: POST /recall → task synthesis → POST /remember → GET /stats. It is a Python CLI script invoked by Claude Code's skill runner.

**Key contracts from Phases 1-4:**
- `POST /recall` request: `{"task_context": str, "top_k": int, "min_tier": str, "tags": list[str]}`
- `POST /recall` response: list of `{id, context, what_worked, what_failed, confidence, tier, tags, times_applied, last_updated}`
- `POST /remember` request: `{"task_context": str, "outcome": str, "patterns_used": list[str]}`
- `POST /remember` response: 202 Accepted
- `GET /stats` response: `{total_patterns, by_tier, avg_confidence, top_tags, recall_hit_rate, recently_promoted}`
