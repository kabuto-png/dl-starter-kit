```python
async def insert_pattern(self, pattern: Pattern) -> None:
    """Insert a new pattern into patterns.jsonl.

    STORE-03: Full read-modify-write cycle held under asyncio.Lock.
    Uses atomic write (NamedTemporaryFile + os.replace) — crash safe.
    Last-write-wins dedup on read (STORE-01): if pattern.id already exists,
    this insert will overwrite the existing record (same as update_pattern).
    """
    async with self._lock:
        patterns = await asyncio.to_thread(self._read_patterns_sync)
        patterns[pattern.id] = json.loads(pattern.model_dump_json())
        await asyncio.to_thread(self._write_patterns_atomic_sync, patterns)
```

Key constraints:
- Method must use `async with self._lock:` — the full read-modify-write cycle under lock (STORE-03)
- Use `asyncio.to_thread` for blocking file I/O (same as update_pattern)
- Convert Pattern to dict using `json.loads(pattern.model_dump_json())` — this handles datetime serialization correctly (model_dump_json serializes datetime to ISO string; json.loads converts back to dict)
- Do NOT use `pattern.model_dump()` directly — datetime fields will be Python datetime objects, not JSON-serializable strings
- Use `self._read_patterns_sync()` and `self._write_patterns_atomic_sync()` — the same private helpers already in the class
- The method does NOT append to confidence_history.jsonl — that is only for confidence updates, not inserts
- Place insert_pattern immediately after (or before) update_pattern in the class body for logical grouping

Do not modify update_pattern, load_active, load_stats, or any other existing method.
Do not change the class constructor or any private helpers.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | insert_pattern concurrent write | mitigate | asyncio.Lock held across full read-modify-write cycle (same as update_pattern in Phase 1); concurrent inserts serialize correctly |
| T-02-08 | Tampering | insert_pattern corrupting existing pattern | accept | Last-write-wins dedup by id (STORE-01): if same id is inserted twice, second write wins; this is the defined behavior, not a corruption risk |
| T-02-09 | Denial of Service | insert_pattern atomic write failure | accept | If os.replace fails (disk full, permissions), exception propagates to distill_and_store outer try/except and is logged; KB is not corrupted (tmp file is not yet renamed) |
| T-02-10 | Information Disclosure | pattern fields in JSONL | accept | JSONL is operator-accessible file; no credentials or API keys stored in Pattern fields; fields are user-submitted task context (not PII mitigation in scope for Phase 2) |
