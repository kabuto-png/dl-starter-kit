<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → POST /remember | User-controlled request body; Pydantic DistillRequest validates before BackgroundTask is spawned |
| FastAPI BackgroundTask → distill_and_store | BackgroundTask runs outside HTTP request scope; exceptions do not propagate back to caller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Denial of Service | POST /remember large request body | accept | FastAPI default request body size limit applies; no custom size limit needed for Phase 2 (hackathon scope) |
| T-02-12 | Denial of Service | BackgroundTask queue overflow | accept | FastAPI BackgroundTasks runs sequentially after response; no queue bounding needed for expected hackathon load |
| T-02-13 | Tampering | DistillRequest body validation bypass | mitigate | Pydantic DistillRequest validates task_context (required str) and outcome (required str) before BackgroundTask is spawned; missing required fields return 422 automatically |
| T-02-14 | Information Disclosure | 422 validation error response | accept | FastAPI 422 responses expose field names and validation messages; this is acceptable for internal hackathon service with no auth surface |
| T-02-15 | Spoofing | patterns_used ID forgery | accept | User can supply any pattern IDs; store.update_pattern silently ignores IDs not found; no auth surface in v1 scope per REQUIREMENTS.md Out of Scope table |
</threat_model>
