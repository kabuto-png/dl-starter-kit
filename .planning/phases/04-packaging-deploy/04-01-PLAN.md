
<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| container process → host filesystem | Container runs as uid 1001 (appuser); volume mount at /app/data limits write scope to that directory only |
| build context → Docker image | .dockerignore controls which host files enter the image; .env exclusion prevents secrets from being baked into the artifact |
| environment → container runtime | AKC_KB_DIR and all credentials injected at runtime by orchestration (AgentBase or docker-compose); not hardcoded in Dockerfile |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | Dockerfile USER | mitigate | RUN useradd -r -u 1001 appuser + USER appuser ensures container process runs as uid 1001, not root; if container is breached, attacker cannot escape to host via root |
| T-04-02 | Information Disclosure | Docker image build | mitigate | .dockerignore excludes .env, .env.*, *.credentials.json — secrets never baked into image artifact; AKC_KB_DIR not in Dockerfile ENV (no runtime config in image) |
| T-04-03 | Tampering | /app/data volume mount | mitigate | /app/data owned by appuser (chown in Dockerfile); appuser is the only user in container — other processes cannot write to KB unless they also run as appuser |
| T-04-04 | Denial of Service | uvicorn --host 0.0.0.0 | accept | Service binds all interfaces in container; network access control is responsibility of AgentBase platform and orchestration layer (firewall, NetworkPolicy) — out of scope for Phase 4 |
| T-04-05 | Repudiation | Container logs (DEPLOY-03) | mitigate | PYTHONUNBUFFERED=1 ensures log lines are flushed immediately; startup log shows KB_DIR path so operators can audit which persistent storage is active after deploy |
</threat_model>
