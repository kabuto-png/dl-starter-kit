The problem & the idea

AI agents relearn the same lessons every session — what worked, what failed, which keyword strategy hit. Nothing compounds.

AKC = shared team memory

Agents recall proven patterns before a task and remember outcomes after. Each pattern carries a confidence score and a tier: gold › production › experimental › demoted.

It gets smarter on its own

Outcomes adjust confidence; a curator promotes proven patterns to gold. The more the team uses it, the better recall becomes — with no manual upkeep by users.

The MCP — the single face

One MCP server wraps AKC, so the same tools work in Claude Code, Claude Desktop, Codex, and Gemini. Hosted on AgentBase (Streamable HTTP) — point a client at the URL, no local install.

Security posture

Trust boundary: open ingestion, authenticated curation. The data plane (recall/remember) is open for low-friction adoption; the control plane (POST /curate) is key-gated — only the Steward can change tiers.

LayerControl
Auth/curate requires X-Curator-Key · constant-time hmac.compare_digest
Integritytier↔confidence invariant — curate clamps confidence into the tier band
Qualityanti-hallucination evals + confidence-gated auto-promotion
Recoverydeterministic image seed → tampering reset by redeploy (~2 min)
AtomicityJSONL writes under lock + tempfile + os.replace (crash-safe)

Verified live: /curate no-key → 401, with key → 200. No secret in the repo (audit clean, 12/12 tests pass).

Demo (5 min) — both planes

① Data plane (users)

In Claude (MCP configured): "Plan a Japan keyword launch." → agent calls akc_recall, surfaces the JP kanji-keyword pattern, plans citing it, then akc_remember records the outcome.

② Control plane (Steward)

In OpenClaw: "Audit the KB." → flags the JP-keyword pattern (production 0.82, applied 18×) as a promote candidate → "Promote it to gold" → /curate → /stats shows it in recently_promoted. Then "show gaps."

Security beat (optional): a no-key /curate → 401 blocked; with key → promoted. The defense runs live.

Roadmap — production hardening

  1. Per-user access via MCP Gateway → Policy Group — principal jwt + condition on the email claim ⇒ restrict to @vng.com.vn. Native to the platform — no custom OAuth code.
  2. Network isolation — AKC as a private agent-runtime-vpc; only the gateway is public → truly "single exposed surface".
  3. Least-privilege IAM — scope the service account to a group with only runtime + registry policies.
  4. Data flywheel at scale — pattern set 30 → 300+ as more teams recall/remember through the one MCP.
